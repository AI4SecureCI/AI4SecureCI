Modules

The AI4SecureCI curriculum is organized into eight connected modules. Module 0 builds the foundation in cloud cyberinfrastructure, AI/ML, and fair AI. Modules 1 through 7 then move students through cloud network security, automated access control, malware detection and classification, adversarial robustness, fairness-aware security, and explainable AI.

Module summary

This table summarizes the main learning question and student-facing output for each module.

Module no. Module title Main question Student output
Module 0 Getting Started with Cloud CyberInfrastructure, Machine Learning, and Fair AI What basic cloud, AI/ML, and fairness knowledge is needed before advanced cloud-security work? A configured cloud environment, basic ML models, and initial bias/fairness analysis using fairness tools.
Module 1 Cloud CI Network Security Is the cloud network showing signs of intrusion or malicious traffic? A cloud network intrusion detection system trained on captured network traffic.
Module 2 Authorization and Automated Access Control in Cloud CI Should a user or process be allowed to access a cloud resource? A deep-learning-based access control model that makes authorization decisions from user/resource metadata.
Module 3 Online Malware Detection in Cloud CI Is this VM or workload behaving maliciously? A defensive ML detector that classifies runtime behavior as benign or malicious.
Module 4 Classifying Malware Threats in Cloud CI What type or family does the detected malware belong to? A malware family classifier trained on behavioral API-call data.
Module 5 Adversarial Attacks and Defenses in Cloud CI Can attackers fool the cloud-security ML model, and how can we defend it? Adversarial examples against malware detectors and a retrained, more robust defensive model.
Module 6 Bias and Fairness in Cloud CI Security Can bias in data or algorithms weaken cloud-security decisions? A bias-audited and bias-mitigated security model using fairness-aware AI methods.
Module 7 Explainable AI for Cloud CI Security Why did the cloud-security ML model make a specific decision? Explainable malware detection results using SHAP-based local and global feature explanations.

Focused comparison: how the modules work together

The modules are not isolated topics. They form a security pipeline: students first build the foundation, then collect cloud-security evidence, train defensive models, test model robustness, improve fairness, and explain the final security decisions.

Focus area Related module(s) Defensive purpose How it connects to the next stage
Foundation and setup Module 0 Builds basic ability in cloud CI, secure configuration, AI/ML workflows, and fairness concepts. Prepares students to understand the data, models, and ethical issues used in all later modules.
Network-level defense Module 1 Looks for suspicious or malicious behavior in cloud network traffic. Provides network evidence that can be used for intrusion detection and broader cloud monitoring.
Access-control defense Module 2 Decides whether users, processes, or requests should be allowed to access cloud resources. Adds authorization intelligence to the defensive pipeline so that access decisions can be evaluated dynamically.
Runtime malware detection Module 3 Determines whether a VM, workload, or runtime process is behaving maliciously. Produces a detection result that can be investigated further through malware family classification.
Threat classification Module 4 Identifies the type or family of malware using behavioral evidence such as API-call data. Turns detection into threat intelligence that can guide mitigation and response.
Adversarial robustness Module 5 Tests whether attackers can evade or manipulate AI-based cloud-security models. Improves the reliability of the detectors and classifiers developed in earlier modules.
Fairness and bias control Module 6 Checks whether data bias or algorithmic bias weakens security decisions or creates exploitable behavior. Makes the security models more trustworthy before they are interpreted and reported.
Explainable security decisions Module 7 Explains why an AI/ML model made a particular security decision. Completes the workflow by turning model output into understandable evidence for students, analysts, and researchers.

Lab workflow

The lab workflow follows the same sequence as the module summary. Each stage produces an artifact that supports the next stage, ending with a complete security report that combines detection, classification, robustness, fairness, and explainability.

Related module(s) Lab activity Student deliverable
Module 0 Configure cloud resources, review secure cloud concepts, practice AI/ML basics, and learn fairness-aware AI concepts. A working cloud/AI foundation that supports the advanced labs.
Modules 1, 2, and 3 Collect or prepare network traffic, access-control metadata, and VM/workload behavior data. Cleaned and structured cloud-security datasets ready for model development.
Modules 1, 2, 3, and 4 Train AI/ML models for intrusion detection, automated access control, malware detection, and malware family classification. Defensive models that answer the main security questions in the module summary table.
Modules 1, 2, 3, and 4 Evaluate trained models using security-relevant metrics such as accuracy, precision, false positives, true positives, and classification performance. A model evaluation report with performance results and supporting graphs or tables.
Module 5 Generate adversarial examples, test whether defensive models can be fooled, and retrain or harden the models. A robustness comparison showing model behavior before and after adversarial defense.
Module 6 Audit datasets and model behavior for data bias and algorithmic bias, then apply mitigation methods. A fairness-aware security model with documented bias findings and mitigation results.
Module 7 Apply explainable AI methods to interpret the final model decisions locally and globally. Explainable results showing which features influenced the model's security decisions.
All modules Combine the setup, data, models, evaluation, adversarial testing, fairness audit, and explanations into one final analysis. A complete cloud CI security report that communicates the problem, method, result, limitations, and defensive value.

Overall learning path: AI4SecureCI guides students from cloud and AI foundations to practical cloud defense, malware analysis, adversarial testing, fairness-aware improvement, and explainable security reporting.
