Modules

The AI4SecureCI curriculum is organized into eight connected modules. Module 0 builds the foundation in cloud cyberinfrastructure, AI/ML, and fair AI. Modules 1 through 7 then move students through cloud network security, automated access control, malware detection and classification, adversarial robustness, fairness-aware security, and explainable AI.

Module summary

Index Module no. Module title Main question Student output
1 Module 0 Getting Started with Cloud CyberInfrastructure, Machine Learning, and Fair AI What basic cloud, AI/ML, and fairness knowledge is needed before advanced cloud-security work? A configured cloud environment, basic ML models, and initial bias/fairness analysis using fairness tools.
2 Module 1 Cloud CI Network Security Is the cloud network showing signs of intrusion or malicious traffic? A cloud network intrusion detection system trained on captured network traffic.
3 Module 2 Authorization and Automated Access Control in Cloud CI Should a user or process be allowed to access a cloud resource? A deep-learning-based access control model that makes authorization decisions from user/resource metadata.
4 Module 3 Online Malware Detection in Cloud CI Is this VM or workload behaving maliciously? A defensive ML detector that classifies runtime behavior as benign or malicious.
5 Module 4 Classifying Malware Threats in Cloud CI What type or family does the detected malware belong to? A malware family classifier trained on behavioral API-call data.
6 Module 5 Adversarial Attacks and Defenses in Cloud CI Can attackers fool the cloud-security ML model, and how can we defend it? Adversarial examples against malware detectors and a retrained, more robust defensive model.
7 Module 6 Bias and Fairness in Cloud CI Security Can bias in data or algorithms weaken cloud-security decisions? A bias-audited and bias-mitigated security model using fairness-aware AI methods.
8 Module 7 Explainable AI for Cloud CI Security Why did the cloud-security ML model make a specific decision? Explainable malware detection results using SHAP-based local and global feature explanations.

Focused comparison: module roles in the security pipeline

The modules are designed as a progression. Students first learn the foundations, then build defensive models, test those models against adversarial behavior, and finally improve trust through fairness and explainability.

Pipeline stage Related module(s) Role in the curriculum Why it matters
Foundation Module 0 Introduces cloud CI, secure cloud configuration, AI/ML basics, and fair/ethical AI concepts. Gives students the background needed to understand the advanced security modules.
Cloud visibility Module 1 Uses network traffic capture, preprocessing, and ML-based intrusion detection. Helps students detect suspicious activity at the cloud network level.
Access decision-making Module 2 Applies deep learning to user/resource metadata for automated authorization decisions. Shows how AI can support dynamic access control in cloud environments.
Malware detection Module 3 Monitors VM or workload behavior and trains ML models to detect malicious activity. Addresses real-time defense against zero-day, polymorphic, and metamorphic malware behavior.
Malware understanding Module 4 Classifies detected malware into types or families using behavioral API-call data. Turns a detection result into actionable threat intelligence and mitigation guidance.
Robustness Module 5 Generates adversarial examples and retrains defensive models to resist evasion. Teaches students how attackers can manipulate AI models and how defenders can harden them.
Fairness and bias control Module 6 Audits and mitigates data bias and algorithmic bias in cloud CI security models. Improves trustworthiness and reduces weaknesses that adversaries may exploit.
Interpretability Module 7 Uses explainable AI methods such as SHAP to interpret model decisions. Helps students explain why a model flagged traffic, access, or malware behavior as risky.

Lab workflow

The lab workflow connects the modules into a practical cloud-security learning path. It starts with setup and data collection, moves into model development, and ends with evaluation, fairness, robustness, and explanation.

Step Activity Related module(s) Expected result
1 Prepare the cloud and AI foundation. Module 0 Students configure cloud resources, learn basic ML workflows, and understand fairness concepts.
2 Collect and prepare cloud security data. Modules 1, 2, and 3 Students work with network traffic, access-control metadata, and VM/workload behavior data.
3 Train defensive AI/ML models. Modules 1, 2, 3, and 4 Students build models for intrusion detection, access control, malware detection, and malware family classification.
4 Evaluate model performance. Modules 1, 2, 3, and 4 Students report accuracy, precision, false positives, true positives, and other relevant security metrics.
5 Stress-test models against adversarial behavior. Module 5 Students generate adversarial examples and improve model robustness through retraining or defense techniques.
6 Audit and mitigate bias. Module 6 Students identify biased data or biased model behavior and apply fairness-aware mitigation methods.
7 Explain model decisions. Module 7 Students use explainable AI to show which features influenced a security decision.
8 Prepare a final security report. All modules Students summarize the problem, data, model, evaluation results, fairness concerns, robustness findings, and explanations.

Overall learning path: AI4SecureCI guides students from cloud and AI foundations to practical defense, adversarial testing, fairness-aware improvement, and explainable security reporting.
