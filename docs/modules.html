Modules

The AI4SecureCI - AI for Secure Cloud Infrastructure curriculum is organized into eight connected modules. Module 0 builds the foundation in cloud cyberinfrastructure, AI/ML, and fair AI. Modules 1 through 7 then move students through cloud network security, automated access control, malware detection and classification, adversarial robustness, fairness-aware security, and explainable AI.

Module summary

This table summarizes the main learning question and student-facing output for each module.

Module no. Module title Main question Student output
Module 0 Getting Started with Cloud CyberInfrastructure, Machine Learning, and Fair AI What basic cloud, AI/ML, and fairness knowledge is needed before advanced cloud-security work? A configured cloud environment, basic ML models, and initial bias/fairness analysis using fairness tools.
Module 1 Cloud CI Network Security Is the cloud network showing signs of intrusion or malicious traffic? A cloud network intrusion detection system trained on captured network traffic.
Module 2 Authorization and Automated Access Control in Cloud CI Should a user or process be allowed to access a cloud resource? A deep-learning-based access control model that makes authorization decisions from user/resource metadata.
Module 3 Online Malware Detection in Cloud CI Is this VM or workload behaving maliciously? A defensive ML detector that classifies runtime behavior as benign or malicious.
Module 4 Classifying Malware Threats in Cloud CI What type or family does the detected malware belong to? A malware family classifier trained on behavioral API-call data.
Module 5 Adversarial Attacks and Defenses in Cloud CI Can attackers fool the cloud-security ML model, and how can we defend it? Adversarial examples against malware detectors and a retrained, more robust defensive model.
Module 6 Bias and Fairness in Cloud CI Security Can bias in data or algorithms weaken cloud-security decisions? A bias-audited and bias-mitigated security model using fairness-aware AI methods.
Module 7 Explainable AI for Cloud CI Security Why did the cloud-security ML model make a specific decision? Explainable malware detection results using SHAP-based local and global feature explanations.
