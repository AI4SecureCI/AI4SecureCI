Modules 3 and 4: from online detection to malware family classification.

The uploaded source explains Modules 3 and 4 as connected defensive learning units. Module 3 asks whether a cloud VM is behaving maliciously. Module 4 asks what type or family the detected malware belongs to.

Module 3: Online Malware Detection in Cloud CI

Main question: Is this cloud VM or workload currently behaving like it is infected?

Students learn AI/ML-assisted malware detection for cloud cyberinfrastructure. The module starts with static and dynamic analysis of cloud-specific malware and explains why signature-only detection is limited against zero-day, polymorphic, and metamorphic malware.

Students learn

  • Static and dynamic malware analysis concepts.
  • Why real-time monitoring is necessary after software has entered a cloud CI environment.
  • How to observe processes, file systems, network activity, memory behavior, and runtime system/API calls.
  • How dynamic features such as performance metrics and time-dependent behavior support online detection.
  • How ML and multimodal/LLM-assisted methods can support detection and contextual interpretation.

Lab 3: Online Malware Detection

Main product: a defensive ML detector that classifies runtime behavior as benign or malicious.

Hands-on workflow

  1. Use an isolated VM environment for controlled malware-behavior collection.
  2. Monitor behavior while the sample executes in the isolated lab.
  3. Capture process, file, network, memory, and API/system-call features.
  4. Preprocess the collected behavior data.
  5. Train an ML model to detect malicious behavior.
  6. Report results with justification, metrics, and graphs.
  7. Use LLM prompts only on sanitized summaries for contextual explanation and triage.

Safety boundary: malware execution should remain isolated. The clean ML/analysis machine should not be directly connected to the dirty malware network without a controlled transfer process.

Module 4: Classifying Malware Threats in Cloud CI

Main question: What kind of malware is this, and which family does it belong to?

Module 4 goes beyond detection. Students learn why classification matters for understanding malware behavior, anticipating likely impact, selecting mitigation strategies, and improving cyber threat intelligence sharing.

Topics covered

  • Malware types such as ransomware, adware, spyware, trojans, and keyloggers.
  • Malware families and behavioral differences between families.
  • Polymorphic and metamorphic malware.
  • Obfuscation, packing, and encryption.
  • ML-based classification and clustering-based approaches.
  • LLM-assisted classification and prompt-engineering pitfalls such as unstructured, multiple, or hallucinated class outputs.

Lab 4: Malware Family Classification

Main product: a family classifier that predicts the malware family or type from behavioral API-call data.

Hands-on workflow

  1. Use a behavioral malware dataset containing API calls from different families.
  2. Preprocess the dataset and prepare features for classification.
  3. Train ML models to classify malware families.
  4. Evaluate model performance for each malware family.
  5. Use prompt engineering to guide or evaluate LLM-based classification from sanitized behavioral data.

Expected skill gain: students learn how to identify malware variants and evaluate classifier effectiveness across different malware families.
