Curriculum details

AI4SecureCI develops cloud CI security training through modules, labs, datasets, and safe practice.

This page summarizes the AI4SecureCI project scope, curriculum design, lab delivery model, technical approach, and safety framing for defensive cloud cyberinfrastructure security education.

Project scope

AI4SecureCI is framed as an advanced cloud cyberinfrastructure security curriculum. The curriculum connects lecture materials with hands-on labs so students develop conceptual understanding and practical skills.

  • Domain: cloud cyberinfrastructure security.
  • Methods: AI/ML-assisted detection, classification, monitoring, and analysis.
  • Learning mode: lectures plus hands-on lab exercises.
  • Data sources: project-created datasets, public datasets, and cloud-service-supported training resources.

Curriculum emphasis

AI4SecureCI emphasizes Modules 3 and 4 as practical cloud-CI malware education: online malware detection, behavioral data collection, family classification, and LLM-supported interpretation.

Static analysis Dynamic analysis Runtime monitoring ML detection API-call classification LLM-assisted reasoning

Technical approach

1. Monitor behavior

Collect process, file-system, network, memory, performance, and API/system-call features from cloud-like workloads.

2. Detect attacks

Train online malware detectors that classify behavior as benign or malicious after software has entered the environment.

3. Classify families

Train models that map malware behavior to known types or families and evaluate per-family model quality.

4. Deploy safely

Use isolated local VMs for risky execution and cloud services for realistic deployment, monitoring, and demonstration.

Cloud and VM design

The strongest lab design is hybrid. Local isolated VMs are safer for malware detonation and behavior capture. AWS EC2-style environments are more realistic for cloud CI deployment, IAM/VPC learning, monitoring, and online detection demonstrations.

Environment Best use Why it matters
VMware / VirtualBox Malware detonation and behavior collection Fast reset, simple isolation, less cloud abuse/billing risk.
AWS EC2 Cloud CI monitoring and detector deployment Realistic IAM, VPC, logging, CloudWatch-style workflows.
Clean ML machine Preprocessing, training, reporting Keeps analysis separate from the dirty malware network.

Safety and responsible use

The site presents malware-related content only for defensive education. Labs should preserve strong isolation, avoid uncontrolled outbound traffic, and use sanitized behavior summaries when applying LLMs.

  • Use clean baselines and snapshots before each run.
  • Use internal/host-only networks for malware labs.
  • Disable shared clipboard, drag-and-drop, shared folders, and USB auto-connect for dirty VMs.
  • Do not connect clean ML analysis systems directly to dirty malware networks unless a controlled transfer process exists.
  • Use cloud environments only with policy-compliant isolation and approval where required.
