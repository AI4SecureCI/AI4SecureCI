AI/ML-assisted cloud cyberinfrastructure security

Advanced AI4SecureCI curriculum for secure cloud CI training.

AI4SecureCI introduces students to cloud cyberinfrastructure security through lecture material and hands-on labs. The revised site highlights online malware detection, malware family classification, safe sandboxing, ML workflows, and LLM-assisted analysis for defensive cybersecurity education.

03

Online Malware Detection

Students learn how cloud VMs and workloads can be monitored for suspicious runtime behavior using static/dynamic analysis, logs, performance features, memory behavior, network activity, and system/API calls.

04

Malware Family Classification

After detection, students learn how to classify malware by type or family using behavioral API-call data, ML models, clustering approaches, and structured LLM-assisted reasoning.

Lab

Safe Hands-on Training

The lab model separates unsafe execution from clean analysis: isolated VMs collect behavior, clean ML machines train models, and cloud services demonstrate realistic deployment and monitoring.

What students build

Collect defensive behavior data

Monitor cloud-like VMs for process, file-system, network, memory, and runtime API activity in an isolated environment.

Train detection models

Preprocess behavioral traces and train ML models that distinguish benign from malicious runtime behavior.

Classify malware families

Use API-call behavior datasets to classify malware into known families and evaluate performance for each family.

Use LLMs responsibly

Use LLM prompts on sanitized summaries to support explanation, triage, and structured classification without exposing unsafe artifacts.

Curriculum focus

The uploaded source describes AI4SecureCI as a cloud CI cybersecurity curriculum with lecture content plus practical labs. Its strongest focus is cloud-focused and cloud-specific malware detection and classification at the VM/workload behavior level.

It does not define the labs as a strict cloud-native malware course centered only on Kubernetes, serverless, container escape, cloud IAM abuse, or control-plane compromise. The site therefore uses careful language: cloud CI malware detection and classification.
